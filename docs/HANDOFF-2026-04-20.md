## 3. 공통 기능

- 정책문서 탭 / 기획문서 탭

- **VoneTableFilter** (`js/components/vone-table-filter.js`) — 그리드 표현식 필터

## 4. 기획문서 (이슈) 매니저 정책

### 4.1 스키마 (`plans-index.json`)
```json
{
  "planId": "VPLAN-### (readonly, draft 는 빈 문자열)",
  "title": "제목",
  "planner": "기획자",
  "issueDate": "YYYY-MM-DD",
  "issueNo": "PROD-##### (Jira/외부)",
  "status": "작성중 | 확정 | 완료 | 보류",
  "priority": "긴급 | 높음 | 보통 | 낮음",
  "slug": "transfer | settle-contract | settle-bill | common",
  "tabId": "페이지 tabId (비어있으면 도메인 공용)",
  "page": "표시명",
  "file": "<YYYY-MM-DD>-<slug>.md",
  "tags": ["..."]
}
```

### 4.2 관리 원칙 (★ 지라 스타일)
- **메타데이터 vs 본문 분리**
  - 메타: `plans-index.json` + `localStorage` 오버레이 (키: `vone:plans:overrides`)
  - 본문: `docs/<slug>/plans/<file>.md` (frontmatter 없는 순수 본문)
- **ID 보호**: `VPLAN-###` 은 편집 불가. 수정 중 날아가지 않음
- **소프트 삭제**: 삭제 버튼 = 상태 `보류` 로 변경 → 기본 리스트 숨김. "보류 포함" 체크로 복원
- **2뎁스 필터링**:
  - Level 1 (도메인): `tabId` 비우고 `slug` 만 지정 → 해당 도메인의 모든 탭에 공용 노출
  - Level 2 (페이지): `tabId` 지정 → 해당 탭에서만 노출
- 활동 로그: 메타 필드 변경 시 자동 기록 (`vone:plans:activity`)
- 댓글: `vone:plans:comments` 에 저장

## 6. 알려진 이슈

- `#planDetailModal` HTML 은 남아있으나 사용되지 않음 (deprecated). 실제 열기 동작은 doc-sidebar 로 통합.
- GitHub Contents API 로 plan md 파일 커밋 시 `vone:gh:token` localStorage 에 PAT 필요 (사이드바 설정 버튼에서 등록)
- 일부 plan md 파일은 아직 본문 없음 (draft, 편집 시 저장 커밋으로 생성됨)
- plans-index.json 과 실제 md 파일 개수 불일치 가능 — 새 md 만든 후 인덱스에 수동 추가 필요 (현재)

## 7. 빠른 검증 체크리스트 (새 컴퓨터에서)

```
1. npx serve . -l 3000
2. http://localhost:3000
3. Firebase Auth 로 @vendys.co.kr 로그인
4. LNB → 이체관리 탭 열림 확인
5. 헤더 "기획문서" 버튼 → 모달 10건 로드 확인
6. 이체관리 페이지 우측 사이드바 "기획문서" 탭 → VPLAN-### 리스트 확인
7. 임의 행 클릭 → 본문 md 로드 확인 (생성 플로우로 안 빠짐)
8. 정산계약 관리 → "상세보기" 클릭 → 정산계약 상세 메타바+8섹션+푸터 확인
9. 정산서 관리 → "상세보기" 클릭 → 정산서 상세 메타바(3 row)+7섹션 확인
10. 메타바/섹션 제목 클릭 → 아코디언 토글 동작 확인
```
