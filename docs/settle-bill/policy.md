# 정산서 관리 정책

> 공통 규칙은 [`docs/common/policy.md`](../common/policy.md) 참조. 이 문서는 **정산서 관리(settle-list)** 페이지에만 해당되는 규칙만 담는다.

## 1. 조건 검색 — 옵션 값

### 1.1 계약서 타입 (multi checkbox)
- 값: `식권대장 / 복지대장 / 퀵대장 / 단체선물대장 / 기타`
- **기본값**: 전체 선택

### 1.2 정산 상태 (multi checkbox)
- 값: `정산 대기 / 정산 확정 / 증빙 완료 / 거래 완료`
- **기본값**: 전체 선택

### 1.3 증빙 수취 유형 (radio)
- 값: `전체 / 공급자 발행 / 시스템 발행 / 종이세금계산서`
- **기본값**: 전체

### 1.4 증빙 상계 / 이체 상계 (radio)
- 값: `전체 / 상계 / 별도`
- **기본값**: 전체

### 1.5 계좌 검증 (radio)
- 값: `전체 / 성공 / 실패`
- **기본값**: 전체

### 1.6 금액 유무 필터 3종 (radio) — 매입 증빙 필요 금액 / 매출 증빙 필요 금액 / 지급 대상 금액
- 값: `전체 / 있음 / 없음`
- **기본값**: 전체
- **의미**: "있음" = 해당 금액 > 0, "없음" = 0 또는 NULL

### 1.7 매입 증빙 상태 / 매출 증빙 상태 (select · 동일 리스트)
- 값: `전체 / 증빙 생성 필요 / 실패 / 완료 / 승인 대기 / 국세청 전송`
- **기본값**: 전체

### 1.8 이체 상태 (select)
- 값: `전체 / 이체 대기 / 이체 승인 / 실행 요청 / 이체 성공 / 이체 실패`
- **기본값**: 전체
- **참고**: 이체관리 페이지의 상태 모델을 공유한다.

## 2. 기간 검색

4종: `정산일자 / 작성일자 / 지급 예정일자 / 전송일자`. 각 필드에 공통 핫키(`오늘 / 이번 달 / 지난 달`) + `date-range`.

## 3. 키워드 검색

5종: `정산서(정산서 IDX) / 계약(계약명·계약 IDX) / 사업자번호 / 거래처(거래처명·거래처 IDX) / 제휴점(제휴점 ID·제휴점명)`.

## 4. 테이블 컬럼 구성

사용자 스펙 순서 유지. 상세 그룹은 docs/settle-bill/plans/ 에 기록.

- Pin Left: `정산서 IDX`
- Pin Right: `정산 상태`, `잠금`, `상세보기`
- 금액 컬럼은 우측 정렬, 날짜 컬럼은 중앙 정렬 (공통 정책 §3, §4)

## 5. 푸터 액션

6버튼 (순서 고정):
1. **정산서 생성** (primary filled) — 일괄 생성 모달
2. **계좌 검증** (primary filled) — 선택 행 대상 계좌 검증 API
3. **이체 생성** (primary filled) — 이체관리로 전송
4. **일괄 잠금** (outline)
5. **잠금 해제** (outline)
6. **일괄 수정** (outline) — 선택 행 다건 편집 모달
