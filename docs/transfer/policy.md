# 이체관리 정책문서

이체관리는 정산서 기반으로 생성된 이체 건을 조회/승인/요청/실행하고, 이체 결과(성공/실패)를 관리하는 페이지입니다.

> 본 문서는 이체관리에 **특화된 정책만** 정리합니다.
> 레이아웃, 디자인 토큰, 공통 UI(라디오/아코디언/LNB/문서 사이드바) 등 공통 정책은 별도 공통 정책문서를 참고하세요.

---

## 1. 조건검색

### 1.1 이체 상태 (라디오)
| 버튼 | data-value | 디폴트 |
|------|-----------|--------|
| 전체 | all | |
| 이체 대기 | wait | **활성** |
| 이체 승인 | approved | |
| 실행 요청 | requested | |
| 이체 실행 | executing | |
| 이체 성공 | success | |
| 이체 실패 | fail | |

### 1.2 거래처 구분 (라디오)
전체(디폴트) / 법인 / 개인

### 1.3 출금 계좌 (드롭다운)
전체 + 13개 출금 계좌, 형식: `서비스명 (은행 계좌번호)`

### 1.4 기간 (3종)
| 필터 | 간편 검색 | 디폴트 |
|------|----------|--------|
| 지급예정일 | 오늘 / 이번 달 / 지난 달 | 이번 달 |
| 생성일자 | 오늘 / 이번 달 / 지난 달 | 없음 |
| 이체요청일 | 오늘 / 이번 달 / 지난 달 | 없음 |

### 1.5 키워드 검색
| 라벨 | placeholder |
|------|------------|
| 이체정보 | 이체관리 IDX |
| 정산서 | 정산서 IDX, 정산서명 |
| 거래처 | 거래처 IDX, 거래처명, 사업자등록 |
| 계약서 | 계약서명 |
| 예금주 | 예금주명 |
| 계좌번호 | 계좌번호 |
| 제휴점 | 제휴점명, 제휴점 SID |

## 2. 요약

5개 카드 1줄 배치. 각 카드: `{금액}원 / {건수}건`.

| 카드 | 색상 강조 |
|------|----------|
| 대기 | 없음 |
| 승인됨 | 보라 (#7c3aed) |
| 요청됨 | 없음 |
| 성공 | 파란 라벨 + 검정 금액 |
| 실패 | 빨강 (#ef4444) |

## 3. 테이블

### 3.1 컬럼 (좌→우)
1. 체크박스 (left 고정)
2. 이체관리 IDX (링크)
3. 정산서 IDX (링크)
4. 정산서명
5. 사업자구분 (개인/법인 텍스트)
6. 사업자명 / 사업자 등록번호 / 대표자명
7. 출금예정 계좌 / 은행명 / 예금주명 / 계좌번호
8. 이체예정금액 (우측정렬)
9. 지급예정일
10. 검증(대표자) / 검증(사업자) / 검증(종합)
11. 거래처 IDX (링크) / 거래처명
12. 제휴점명
13. 통장거래 IDX (이체 성공 시에만, 링크)
14. 생성일시/생성자, 요청일시/요청자, 지급일시, 수정일시/수정자
15. 실패 사유
16. 상태 (right 고정)
17. 상태 변경 (right 고정) — 행별 액션 버튼

### 3.2 상태 색상 (텍스트만, 배경 없음)
| 상태 | 색상 |
|------|------|
| 이체 성공 | #16a34a |
| 이체 실패 | #ef4444 |
| 이체 대기 | #d97706 |
| 이체 승인 | #7c3aed |
| 실행 요청 | #2563eb |

### 3.3 행별 액션 (상태 변경 컬럼)
| 상태 | 버튼 |
|------|------|
| 이체 실패 | `성공 처리` (stroke) + `이체 확인` (filled) |
| 이체 성공 | `실패 처리` (disabled) |
| 이체 대기 / 이체 승인 | `정보갱신` (filled) |

### 3.4 예금주 검증 로직
1. 양쪽 문자열 괄호/공백 제거
2. 완전 일치 → 100%
3. 한쪽 포함 → 80%
4. 글자 단위 매칭 → (일치 글자 수 / 긴 문자열 길이) × 100
- 60% 미만: 빨간색 + bold
- 종합: `{일치율}% ({기준})` — 개인=대표자, 법인=사업자

### 3.5 행 강조
- 이체 실패 행: `#fff8f8` 연한 빨간 배경

## 5. 성공처리 (모달)

이체 실패 행의 `성공 처리` 버튼 클릭 시 표시.

### 5.1 이체 정보 (아코디언)
디폴트 열림. 6개 필드를 readonly input으로 표시:
- 이체 IDX, 정산월, 정산서 IDX, 정산계약명, 이체예정금액, 제휴점명

### 5.2 조건 검색 (아코디언)
| 필드 | 타입 | 디폴트 |
|------|------|--------|
| 거래일시 | datetime-local 범위 | 해당 일자 00:00 ~ 23:59 |
| 금액 | number 범위 | `이체예정금액 ~ 이체예정금액` |
| 입출금 | 고정 텍스트 `입금` | 변경 불가 |

- 헤더 액션: `필터 초기화` + `데이터 조회`

### 5.3 통장 거래내역 그리드
- 단일 선택 (`rowSelection: 'single'`)
- 선택 1건일 때만 `성공 처리` 활성화

### 5.4 성공 처리 확정 플로우
`성공 처리` 클릭 → 3-버튼 커스텀 다이얼로그 표시
- `닫기` → 아무 동작 없이 닫기
- `정산서도 연결 후 성공처리` → 정산서 IDX에 통장거래 연결 + 성공 처리
- `그냥 성공처리` → 정산서 연결 없이 이체 건만 성공 처리

메시지 형식:
```
정산서IDX: {settleIdx} 에 해당 통장내역(IDX {idx})을
연결하여 성공 처리합니다.
정산서도 연결하시겠습니까?

연결금액: {amt}원
연결가능금액: {amt}원
```

### 5.5 닫기
X 버튼 / 취소 버튼 / 오버레이 클릭

## 8. 기타

### 8.1 그리드 툴바
| 요소 | 위치 | 설명 |
|------|------|------|
| 전체 건수 | 좌측 | `전체 : {N}건` |
| 컬럼 초기화 | 우측 | 컬럼 상태 리셋 |
| 페이지 단위 | 우측 | 1,000 / 2,000 / 5,000 / 10,000 / 20,000개씩 |
| 페이지네이션 | 우측 | `< 1 2 3 4 5 ... 8 >` |

### 8.2 AG Grid 특화 설정
- rowHeight/headerHeight: 28px
- suppressRowClickSelection: true
- 필터 아이콘: 헤더 호버 시에만 표시
- 헤더 우클릭: Pin Left/Right, No Pin, Auto Size (All), Reset Columns
- 그리드 내 텍스트: 배경색 태그 금지 (순수 텍스트 / 색상 텍스트만)
