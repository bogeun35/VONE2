# 성공처리 모달 3버튼 플로우

## 배경

이체 실패 건을 수기로 성공 처리할 때, 통장 거래내역 연결(정산서 IDX ↔ 통장거래 IDX)을 함께 수행해야 하는 경우와, 단순히 이체 상태만 성공으로 바꿔야 하는 경우가 섞여 있어 단일 확인 다이얼로그로는 의도를 구분할 수 없었음.

## 요구사항

- `성공 처리` 클릭 시 즉시 커밋하지 말고 **3-버튼 다이얼로그** 표시
  - `닫기` — 아무 작업 없음
  - `정산서도 연결 후 성공처리` — 통장거래 연결 + 이체 성공 처리 (2단계)
  - `그냥 성공처리` — 이체 상태만 성공으로 전환
- 다이얼로그 본문: 정산서 IDX, 통장거래 IDX, 연결금액, 연결가능금액 표시

## 구현

- `transfer.js:confirmSuccessChoice()` — 오버레이 DOM 주입 방식
- 디자인 시스템 준수: 버튼 3종 모두 파랑 계열 (닫기=stroke, 두 액션=filled)
