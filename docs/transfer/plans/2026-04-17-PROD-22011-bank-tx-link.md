# 이체관리 개선 안

## 배경

* 실제 이체 프로세스를 1Cycle 시행하면서 부족한 부분과 대해서 기능을 추가하여, 관리의 용이성을 증가시키고자 함.

## 요구사항1 (이체 테이블)

* **상태값에 대한 정의 추가**
    * 이체 대기 : 정산서에서 이체 생성 시
    * 이체 승인 [추가] : 이체 대기 인 경우 특정 사람이 승인으로 변경 할 수 있다
    * 실행 요청 [추가] : 이체 실행 Task를 진행 요청 시 상태를 실행 요청으로 변경된다.
    * 이체 실행 : 이체가 더존에 task 진행된 상태 (약 1-8초)
    * 이체 성공 : 이체가 더존에서 성공 시
    * 이체 실패 : 이체가 더존에서 실패 시
<br>
* **컬럼 추가**
    * 이체 승인일시
    * 이체 승인자
    * 통장거래내역IDX
    * 일괄작업현황IDX

## 요구사항2 (API 추가)

* **API 추가 [이체 승인하기]**
    * 이체 대기에 대해서 승인으로 바꾸는 API 추가 [사용자 권한 상 별도 필요]
* **API 추가 [이체 갱신하기]**
    * 이체 대기, 승인인 경우 예금주명이나 거래처의 데이터가 변경되는 경우, 이를 적용하는 기능
        * 정산서에서 가져 올 수 있는 정보를 기반으로 금액을 포함하여, 갱신이 가능
        * 이체 승인 및 이체 대기 인 경우 정산서의 예금주명, 은행명, 은행코드, 계좌번호, 지급 예정일을 갱신한다.
            * 필수 파라미터 : 이체 IDX

## 요구사항3 (API 수정)

* **API수정 [이체 삭제]**
    * 이체 삭제의 조건 : 이체 대기+ 이체 승인 상태 일 때 가능
* **API수정 [이체 요약]**
    * 승인됨 컬럼(금액,수량)을 추가한다.
* **API수정 [이체 생성]**
    * 이체 금액이 들어 간 경우 현재 지급 예정금액-지급 완료 금액-이체(실패를 제외한 sum) 에 맞춰서 검증을 한다.
* **API수정 [성공 처리]**
    * 성공처리 시 payload에 bankidx를 받을 수 있다. 이때는 실패 사유에 bankIdx = 123211를 넣고 성공처리한다.
* **API수정 [이체 조회]**
    * 계약서 타입 필터추가(멀티)
    * 정산년월 추가
    * 계약구분 추가
    * 정산서 태그 추가
    * 대표자명 추가
    * 출금예정계좌번호 추가
    * 검증(대표자) 추가 : 
    * 검증(사업자) 추가
    * 검증(%)추가
        사업자가 개인이면 대표자 사업자가 법인이면 사업자 검증으로 노출
        * 변수 3개
            예금주명 (ownerName)
            대표자명 (ceoName)
            사업자명 (bizName)
            검증(대표자) = matchRate(예금주명, 대표자명)
            검증(사업자) = matchRate(예금주명, 사업자명)
            검증(%) = 사업자구분 === '개인' ? 검증(대표자) : 검증(사업자)
            matchRate 로직
            완전 일치 → 100%
            한쪽이 다른 쪽을 포함 → 80%
            그 외 → (일치 글자수 / 긴 쪽 길이) × 100 (반올림)
            둘 중 하나가 빈 값 → 0%
            비교 전 괄호·공백은 제거합니다.
    * 통장거래내역IDX 컬럼 변경 -> JOIN에서 컬럼으로
    * 일괄 처리IDX추가

## 요구사항4 (프론트)

* **조건검색 추가**
    * 조건검색에 계약서 타입 (CommonCode) 추가
    * 이체상태 라디오 버튼형태로 변경 (CommonCode)
        * \[전체\]\[이체 대기\]\[이체 승인\]\[실행 요청\]\[이체 실행\]\[이체 성공\]\[이체 실패\]
        * 디폴트는 이체대기
    * 거래처 구분도 라디오버튼형 UI로 구성
    * 간편검색기능을 생성일자, 이체 요청일에도 추가
        간편검색에서 전체 제거
* **요약**
    * 요약에 이체 승인 추가(5단)
        * 요약의 순서 변경 건수 -> 금액
* **Table**
    * 정산연월 추가
    * 계약 구분추가
    * 지급예정일 컬럼추가
    * 대표자명 추가
    * 사업자명 추가
    * 사업자 구분 추가(개인,법인)
    * 검증(대표자) 
    * 검증(사업자)
    * 검증(%)
    * 이체IDX 리프트 핀
    * 정산년월추가
* **성공 처리**
    * 처리 전 모달이 뜬다
    * 모달은 거래일시 \~ 거래일시 / 금액 \~ 금액 을 입력 할 수 있다
    * 디폴트는 오늘 금액은 해당 이체의 금액을 넣어두고 조회가 된다
    * 입출금 구분은 입금으로만 조회한다.
* **정보 갱신**
    * 상태가 이체대기 이체승인인경우 상태변경란에 정보갱신 버튼이 노출
    * 클릭 시 정보갱신 API를 호출
* **선택 정보 갱신**
    * 루프로 처리하는 기능
