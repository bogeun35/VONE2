# 통장거래 IDX 컬럼 추가

## 배경

* 실제 이체 프로세스를 1Cycle 시행하면서 부족한 부분과 대해서 기능을 추가하여, 관리의 용이성을 증가시키고자 함

## 요구사항1 (이체 테이블)

**상태값에 대한 정의 추가**

* 이체 대기 : 정산서에서 이체 생성 시
* 이체 승인 [추가] : 이체 대기 인 경우 특정 사람이 승인으로 변경 할 수 있다
* 실행 요청 [추가] : 이체 실행 Task를 진행 요청 시 상태를 실행 요청으로 변경된다.
* 이체 실행 : 이체가 더존에 task 진행된 상태 (약 1-8초)
* 이체 성공 : 이체가 더존에서 성공 시
* 이체 실패 : 이체가 더존에서 실패 시
<br>

**컬럼 추가**

* 이체 승인일시
* 이체 승인자
* 통장거래내역IDX
* 일괄작업현황IDX

* **API수정[요약]**
    * 승인됨 컬럼(금액,수량)을 추가한다.

    * 연결 완료 금액을 추가하여 검증을 추가한다.

    * 이체 금액이 들어 간 경우 현재 지급 예정금액-지급 완료 금액-이체(실패를 제외한 sum) 에 맞춰서 검증을 한다.
