# VONE2 공통 정책문서

> 페이지 전반에 공통으로 적용되는 **UX / UI / 데이터 표현 / 그리드** 규칙.
> 특정 페이지에만 해당되는 규칙은 각 페이지 정책문서(`docs/<page>/policy.md`)에 둡니다.

> ⚠️ **작업 원칙 (매우 중요)**
> - **임의 값 금지**. 숫자, 색상, 단위, 단계, 기본값 등은 모두 아래 정책을 따른다.
> - 정책에 없으면 **반드시 사용자에게 물어보고**, 답변을 받으면 **이 문서에 추가**해서 다음 작업 때 재활용한다.
> - 정책이 바뀌었으면 이 문서부터 수정한 뒤 코드에 반영한다.

---

## 1. 버튼 디자인 시스템

- **3색**만 사용: 파랑 / 빨강 / 검정
- **Filled(primary)** = 실행/이동 (저장·조회·승인·이체)
- **Stroke(outline)** = 모달 오픈 (필터 초기화·컬럼 선택·신규 생성 등)
- **빨강** = 삭제 **전용** (`btn-outline-danger`, `btn-danger`)
- **검정** = input 옆 검증/검색 보조 버튼 · **프론트 전용 핫키** (ex: 날짜 검색의 `오늘·이번 달·지난 달`, `.btn-period`)

## 2. 그리드 페이지네이션

| 항목 | 값 |
| --- | --- |
| 페이지 사이즈 옵션 | `1000 / 2000 / 5000 / 10000 / 20000` |
| 기본 페이지 사이즈 | `1000` |
| 페이지 버튼 표시 | prev · window(현재 ±2) · ellipsis · last · next |
| 페이지 번호 시작 | 1 (UI 기준 / API 는 0-based) |

- 모든 조회 테이블은 **동일 옵션**을 공유한다. 페이지 다시 정의 금지.
- AG Grid 를 쓰는 경우 `pagination: true` + `suppressPaginationPanel: true` 로 두고, 커스텀 페이지 바를 사용한다.

## 3. 데이터 표현

| 데이터 | 표시 규칙 |
| --- | --- |
| NULL | **빈 셀** — "NULL" 등 라벨 표시 금지 |
| 빈 문자열 | 공백 (렌더하지 않음) |
| Y/N | Y=파랑 굵게, N=회색 |
| 숫자 | `tabular-nums`, 우측 정렬 |
| 비율 | `(value*100).toFixed(2) + '%'` |
| 날짜 | `YYYY-MM-DD` (input 은 `lang="sv-SE"`) |
| 금액 | 3자리 콤마, 화폐기호 없음 |

## 4. 그리드 컬럼 폭 (기본값)

- IDX 계열: `40~44`
- Y/N / 짧은 라벨: `50~60`
- 날짜: `82`
- 금액: `92` 이상
- 일반 텍스트: `70~110`
- 우측 고정(액션 버튼): `72~82`
- **원칙**: 모든 컬럼이 한 화면에 들어오도록 **타이트**하게. 사용자가 리사이즈 후 템플릿으로 저장.
- **AG Grid `defaultColDef.minWidth = 40`** 로 설정 — AG Grid 내부 기본값(100)이 너무 커서 IDX 같은 좁은 컬럼이 40으로 줄어들지 않는다.
- AG Grid 셀/헤더 공통 패딩: `--ag-cell-horizontal-padding: 5px`, `--ag-grid-size: 4px` (style.css 에 정의됨). 페이지별로 재정의 금지.

### 4.1 헤더 우클릭 컨텍스트 메뉴 (필수)
- 모든 AG Grid 는 `GridColumnContext.attach({ gridDiv, gridApi })` 호출로 공통 컨텍스트 메뉴를 붙인다
- 제공 액션: **Pin Left / Pin Right / No Pin · Auto Size / Auto Size All · Reset Columns**
- AG Grid Community 에는 기본 제공되지 않으므로 공통 컴포넌트(`js/components/grid-column-context.js`) 를 반드시 연결

## 5. 링크 / 액션 셀

- IDX 컬럼 등 "다른 페이지로 이동"이 가능한 값은 `class="grid-link"` 로 렌더
- 클릭 시 해당 **상세 탭**으로 이동. 상세 탭은 **고정 id** (`{domain}-detail`)
  - 같은 도메인은 단일 탭에서 **덮어쓰기** — idx 별 개별 탭을 만들지 않는다
  - 예: `settle-contract-detail`, `shop-detail`, `partner-detail`
- 탭 제목에 식별자 표기: `정산계약 상세 #21949`

## 6. 필터 행 구조

- 필터 행은 **조건 검색 / 날짜 검색 / 키워드 검색** 3행을 기본으로 함
- 필드는 한 행 안에 `flex-wrap` 으로 흘려서 배치. 임의로 행을 늘리지 않는다.
- 필터 헤더 우측 액션: **필터 초기화(outline) + 데이터 조회(filled primary)** 고정

### 6.1 기본값
- **계약 상태 / 사용여부 / 유효여부** 류 라디오는 `유효` 등 "살아있는 값" 을 기본으로
- **타입 체크박스 멀티(계약서 타입 등)**: 기본 **전체 선택**
- **전체**가 있는 라디오는 그 외 요건이 없으면 `전체`

### 6.2 필터 리셋
- **전체 비움이 아니라** — 위의 기본값으로 복원

## 7. 탭 / LNB

- LNB 메뉴 번호는 **순서 기반 CSS counter**. HTML/JS 에 하드코딩 금지
- 번호는 LNB(사이트맵)에만 노출. 페이지 헤더/브레드크럼/탭에는 숫자 X
- 탭 상태는 localStorage. 고정 탭은 `closeAll` 에서 유지
- 상세/신규 탭은 `detailOf` 로 부모를 명시 — 브레드크럼 자동 구성

## 8. 문서 사이드바

- 각 페이지의 `data-doc="<slug>"` 에 따라 `docs/<slug>/policy.md` 와 `docs/<slug>/plans/*.md` 를 로드
- 정책문서 탭 / 기획문서 탭 두 개
- 편집: Toast UI MD 에디터 + Ctrl+V 이미지 자동 업로드 (`docs/assets/YYYY-MM/`)
