# 이체관리 정책문서

## 1. 페이지 개요

이체관리는 정산서 기반으로 생성된 이체 건을 조회/승인/요청/실행하고, 이체 결과(성공/실패)를 관리하는 백오피스 페이지입니다.

- 경로: 정산/청구 > 이체관리
- 파일 구조:
  - `index.html` - 페이지 마크업
  - `css/style.css` - 전체 스타일
  - `js/app.js` - 공통 기능 (LNB, 필터 아코디언, 문서 사이드바)
  - `js/pages/transfer.js` - 이체관리 전용 로직 (AG Grid, 모달)
  - `docs/transfer.md` - 기획문서 (버전별 변경사항)
  - `docs/transfer-policy.md` - 정책문서 (본 문서)

## 2. 전체 레이아웃 구조

```
┌─────────────────────────────────────────────────┐
│  Header (60px, fixed)                           │
├────────┬────────────────────────────┬────────────┤
│  LNB   │  Content                   │ Doc Sidebar│
│ 180px  │  (flex: 1)                 │  (420px)   │
│        │  ┌─ Page Header ──────┐    │  열기/닫기 │
│        │  ├─ Filter Box ───────┤    │            │
│        │  ├─ Summary Box ──────┤    │            │
│        │  ├─ Grid Toolbar ─────┤    │            │
│        │  ├─ AG Grid ──────────┤    │            │
│        │  │  (flex: 1)         │    │            │
│        │  ├─ Grid Action Footer┤    │            │
│        │  └────────────────────┘    │            │
└────────┴────────────────────────────┴────────────┘
```

### 레이아웃 규칙
- `html, body`: `height: 100%; overflow: hidden` - 페이지 자체 스크롤 없음
- Header: `position: fixed; height: 60px`
- Layout: `height: calc(100vh - 60px); display: flex`
- Content: `flex: 1; overflow: hidden; display: flex; flex-direction: column`
- AG Grid: `flex: 1; min-height: 0` - 남은 공간을 모두 채움 (반응형 높이)
- Grid Action Footer: `flex-shrink: 0` - 그리드 하단에 항상 표시

## 3. 필터 영역

### 3.1 아코디언 동작
- 접기/펼치기 가능 (`.filter-header` 클릭)
- 디폴트: 열림 상태 (`open` 클래스)
- 애니메이션: `max-height` 트랜지션 (0.25s)

### 3.2 조건 설정

#### 이체 상태 (라디오 버튼 그룹)
| 버튼 | data-value | 디폴트 |
|------|-----------|--------|
| 전체 | all | |
| 이체 대기 | wait | **활성** |
| 이체 승인 | approved | |
| 실행 요청 | requested | |
| 이체 실행 | executing | |
| 이체 성공 | success | |
| 이체 실패 | fail | |

#### 거래처 구분 (라디오 버튼 그룹)
| 버튼 | data-value | 디폴트 |
|------|-----------|--------|
| 전체 | all | **활성** |
| 법인 | corp | |
| 개인 | personal | |

#### 출금 계좌 (드롭다운)
- 전체 + 13개 출금 계좌 옵션
- 형식: `서비스명 (은행 계좌번호)`

### 3.3 기간 설정
3개의 날짜 필터, 각각 간편 검색 버튼 포함:

| 날짜 필터 | 간편 검색 | 디폴트 |
|----------|----------|--------|
| 지급예정일 | 오늘 / 이번 달 / 지난 달 | 이번 달 활성 |
| 생성일자 | 오늘 / 이번 달 / 지난 달 | 없음 |
| 이체요청일 | 오늘 / 이번 달 / 지난 달 | 없음 |

- 날짜 입력: `<input type="date">` 2개 (시작 ~ 종료)

### 3.4 키워드 검색
| 필드 | placeholder |
|------|------------|
| 이체관리 IDX | 이체관리 IDX |
| 정산서 IDX | 정산서 IDX, 정산서명 |
| 거래처 | 거래처 IDX, 거래처명, 사업자등록 |
| 계약서명 | 계약서명 |
| 예금주명 | 예금주명 |
| 계좌번호 | 계좌번호 |
| 제휴점 | 제휴점명, 제휴점 SID |

### 3.5 필터 헤더 액션 버튼
- **필터 초기화**: 모든 필터를 기본값으로 리셋
- **데이터 조회**: 검색 실행 (파란 버튼 + 돋보기 아이콘)

## 4. 요약 영역

5개 카드가 가로 배치됨 (1줄):

| 카드 | CSS 클래스 | 색상 강조 |
|------|-----------|----------|
| 대기 | (기본) | 없음 |
| 승인됨 | summary-approved | 보라색 (#7c3aed) |
| 요청됨 | (기본) | 없음 |

- 각 카드: `{금액}원 / {건수}건` 형식
- 금액은 `<strong>` 태그로 강조

### 5.3 셀 렌더러

#### StatusRenderer
상태값에 따라 텍스트 색상만 변경 (배경색 없음):
| 상태 | 색상 |
|------|------|
| 이체 성공 | #16a34a (초록) |
| 이체 실패 | #ef4444 (빨강) |
| 이체 대기 | #d97706 (노랑) |
| 이체 승인 | #7c3aed (보라) |
| 실행 요청 | #2563eb (파랑) |

#### BizTypeRenderer
사업자구분을 순수 텍스트로 표시 (배경색 태그 없음). 값: `개인` 또는 `법인`

#### LinkRenderer
텍스트를 파란색 링크 스타일로 표시 (`<a class="grid-link">`)

#### AmountRenderer
숫자를 `toLocaleString()`으로 천 단위 콤마 표시. 우측 정렬.

#### MatchRenderer
예금주 검증 일치율을 퍼센트로 표시:
- 60% 이상: 초록색 텍스트
- 60% 미만: 빨간색 텍스트 + bold

#### MatchTotalRenderer
종합 검증을 `{일치율}% ({기준})` 형식으로 표시:
- 개인사업자: 대표자 기준
- 법인사업자: 사업자 기준

#### ActionRenderer
상태에 따라 다른 액션 링크 표시:
| 상태 | 액션 |
|------|------|
| 이체 실패 | `성공 처리` (초록) + `이체 확인` (파랑) |
| 이체 성공 | `실패 처리` (회색) |
| 이체 대기 | `이체 요청` (파랑) |

### 5.4 예금주 검증 로직 (calcMatchRate)
```
1. 양쪽 문자열에서 괄호, 공백 제거
2. 완전 일치 → 100%
3. 한쪽이 다른쪽을 포함 → 80%
4. 글자 단위 매칭 → (일치 글자 수 / 긴 문자열 길이) × 100
```

### 5.5 행 스타일
- 이체 실패 행: 연한 빨간 배경 (`#fff8f8`)

### 5.6 컬럼 우클릭 메뉴
AG Grid 헤더 우클릭 시 커스텀 컨텍스트 메뉴 표시:
- Pin Left / Pin Right
- No Pin
- Auto Size / Auto Size All
- Reset Columns

### 5.7 AG Grid UI 규칙
- 필터 아이콘: 헤더 호버 시에만 표시 (컬럼 텍스트 짤림 방지)
- 체크박스: 중앙 정렬 (세로/가로)
- 그리드 내 텍스트: 배경색 태그 사용 금지, 순수 텍스트 또는 색상 텍스트만 사용

## 8. 성공 처리 모달

이체 실패 건의 `성공 처리` 클릭 시 표시:

### 모달 정보 표시
- 정산서명: 해당 행의 `settleName`
- 이체예정금액: 해당 행의 `transferAmount` (콤마 포맷 + 원)

### 모달 폼 필드
| 필드 | 타입 | 디폴트 |
|------|------|--------|
| 거래일시 | date 범위 | 오늘 ~ 오늘 |
| 금액 범위 | number 범위 | 해당 이체 금액 ~ 해당 이체 금액 |
| 입출금 구분 | 고정 텍스트 | `입금` (변경 불가) |

### 모달 액션
- 취소: 모달 닫기
- 조회: 통장 거래내역에서 매칭 건 검색

### 닫기 방식
- X 버튼 / 취소 버튼 / 오버레이 클릭

## 9. 문서 사이드바

### 열기/닫기
- 페이지 헤더의 `기획문서` 버튼 클릭으로 토글
- 닫기: X 버튼 또는 기획문서 버튼 재클릭
- 너비: 420px, 트랜지션 애니메이션 (0.3s)

### 탭 구조
| 탭 | 파일 | 설명 |
|------|------|------|
| 정책문서 | `docs/{pageName}-policy.md` | 전체 동작/규칙 설명 |
| 기획문서 | `docs/{pageName}.md` | 버전별 변경사항 |

### Markdown 렌더링
- `marked.js` 라이브러리 사용
- Frontmatter 파싱: `version`, `lastUpdated`, `author`, `status`
- 버전 바: 버전 배지 + 상태 배지 + 날짜 + 작성자
- 상태별 색상:
  - 작성중: 노란 배경
  - 검토중: 파란 배경
  - 확정: 초록 배경
- 변경이력 섹션: 접기/펼치기 토글 (디폴트: 접힘)
